loginsysteem met php/mysql

TOETSINFORMATIE

DATUM: 18-11-2014

Opdracht:

Het maken van een inlogsysteem voor het bedrijf Classic Car Models.

Beschrijving:

Je krijgt als Webdeveloper de opdracht binnen om het prototype van een inlogsysteem te maken voor het bedrijf Classic Car Models. CCM wilt graag een waterdicht systeem hebben, waarbij op basis van rol binnen het bedrijf, restricties zijn. Deze restricties zijn hieronder te vinden.

Er zijn drie rollen waar ingelogd op kan worden:

  • Admin: Mag overal in het systeem komen
  • Sales: Komt alleen op de 'sales' pagina
  • Finance: Komt alleen bij 'finance' pagina

Criteria

  • index.php in de root wordt gebruikt als router naar de pagina's
  • Alle inlog afhandelingen in de authController.php
  • Databaseconnectie in config/database.php
  • Inloggen met email en password

Form validatie

  • Email moet een email format hebben als input (zowel front-end als back-end)
  • Melding als één van velden niet is ingevuld (zowel front-end als back-end.)

Form sanitizing

  • Zorg dat spaties voor en na inloggen worden weggehaald
  • Berichten uit de url veilig maken mbv stripslashes of FILTER_SANITIZE_ENCODED
  • Prepared statements gebruiken om user input te behandelen ter voorkoming van sql injectie

Messaging

  • Foutafhandeling bij verkeerd inloggen. ("Gebruikersnaam en/of wachtwoord verkeerd") naar inlog.php via index.
  • confirmatie bij juist inloggen ("Welkom < firstName > , uitloggen? (uitloggen moet een link zijn naar uitlogscript) ");
  • foutafhandeling bij niet ge-autoriseerd bezoek pagina ("Permission denied") -> naar < rol >.php via index
  • confirmatie bij uitloggen ('u bent succesvol uitgelogd') -> naar inlog.php via index

Authorisatie

  • Er wordt een session aangemaakt
  • Correcte check of username password gelijk zijn aan item uit database
  • Uitlogscript aanwezig

Bonus

  • Hou bij hoe vaak de user fout heeft ingelogd
  • indien iemand 3x fout heeft ingelogd, moet er een link in het form komen met 'Wachtwoord vergeten?'; Hoeft verder niet specifiek ergens naar te linken
  • indien iemand 5x fout heeft ingelogd, moet er een link komen met ('te vaak foutief ingelogd, neem contact op met administrator') Inloggen moet nu onmogelijk worden gemaakt. (niet alleen via front-end).
  • Voeg een 403 (permission denied) foutmelding aan de header toe als men ergens komt zonder authorisatie.
  • Maak gebruik van classes om je code te organiseren
